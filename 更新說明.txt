Kitchen OS v3.1 Item Master 更新檔

請將以下 3 個檔案放到 kitchen-calculator 最外層：
1. item-master.js（新增）
2. item_master.json（新增）
3. sw.js（覆蓋原本的 sw.js）

不需要刪除 daily.html、import.html 或其他檔案。
舊的 item-id-fix.js 可以保留，但新版不再依賴它。

本次修正：
- 依「央廚次序」建立固定品項 ID：CM001～CM103。
- 到貨驗收完成後，同時儲存 valuesById。
- 每日盤點改用央廚次序／Item ID 取得今日到貨。
- 品名只作畫面顯示與舊資料備援，不再作主要關聯鍵。
- 舊驗收資料若有 sourceRows，開啟頁面後會自動補建 ID 資料。

巧克力蛋糕：
央廚次序 21 → CM021
即使 Excel 品名與盤點品名規格文字不同，仍會帶入同一筆到貨。

更新步驟：
1. 把 3 個檔案上傳到 GitHub 專案根目錄；sw.js 選擇覆蓋。
2. 等 GitHub Pages 更新約 1～3 分鐘。
3. 將網站所有分頁完全關閉。
4. 重新開啟網站並重新整理兩次，讓新版 Service Worker 接管。
5. 先進「央廚到貨驗證」，載入該日已存應到／驗收資料。
6. 再進「每日盤點」確認今日到貨。

正確範例：
昨日盤點 4.6 + 今日到貨 6 = 盤點前應有 10.6
今日總計 8.7，推算使用量為 1.9 模。
