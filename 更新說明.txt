Kitchen OS 品項固定 ID 更新 v3.0.0-beta.1

請將以下兩個檔案上傳到 GitHub 專案最外層（與 index.html 同一層）：
1. item-id-fix.js（新增檔案）
2. sw.js（覆蓋原檔）

更新內容：
- 依 6601 的「央廚次序」建立固定 Item ID：CM001～CM103。
- 到貨驗收完成後，同時儲存 valuesById，不再只用品名。
- 每日盤點優先依 Item ID 讀取今日到貨。
- 舊的已驗收資料若含 sourceRows，也會自動補建 Item ID。
- 仍保留名稱對應作為舊資料備援。

更新後操作：
1. 等待 GitHub Pages 約 1～3 分鐘。
2. 關閉網站分頁後重新開啟。
3. 再重新整理一次，讓新版 Service Worker 接管。
4. 重新進入「央廚到貨驗證」載入 7/24 已存應到，再進每日盤點確認。
