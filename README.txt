Kitchen OS v1.3｜隔日到貨驗證測試版

本版新增：
1. 今天下完貨後，可上傳叫貨 Excel，選擇「預計到貨日」（預設明天）。
2. 按「儲存應到（隔日驗證）」後，資料會依到貨日期儲存在 Firebase。
3. 隔天收貨時，選擇今天日期，系統自動載入前一天儲存的應到數量。
4. 填寫實到數量；有差異必須選擇原因，確認後才同步給每日盤點。
5. 原 v1.2 的每日盤點、主管總表及自動填入 6601 叫貨表功能均保留。

建議測試流程：
A. 今天在「央廚到貨驗證」上傳下完貨的 Excel。
B. 確認預計到貨日為明天，按「儲存應到（隔日驗證）」。
C. 將日期改成明天，按「重新載入此日應到」，模擬隔天收貨。
D. 填寫實到，有差異時選原因，再按「確認並同步今日實到」。
E. 進入每日盤點，確認今日到貨已帶入「實到數量」，而非原本應到數量。

注意：此為測試版。請先使用副本與少量品項測試，確認 Excel 辨識欄位符合實際叫貨表。
