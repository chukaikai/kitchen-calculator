Kitchen OS v1.1｜盤點與到貨驗證測試版

新增：
1. 央廚 Excel 匯入後顯示「應到／實到／差異／原因」。
2. 有到貨差異時，必須選擇原因才能同步。
3. 到貨資料同步至 Firebase Firestore，所有裝置共用。
4. 央廚總表新增「複製叫貨表數字」：依 1～103 固定順序，只複製數字，一次貼入叫貨表單一欄。
5. 複製前會檢查漏盤與負使用量；有異常時先提示主管確認。

測試方式：
A. 首頁 → 央廚到貨驗證 → 上傳 Excel → 填實到 → 確認同步。
B. 各站完成盤點。
C. 央廚盤點總表 → 複製叫貨表數字 → 貼入 Excel/Google Sheet 的第一格。

注意：Excel 欄位辨識仍需用正式叫貨檔測試。請先用副本，不要直接覆蓋正式叫貨表。
