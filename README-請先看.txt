Kitchen OS v2.0 Beta｜6601 到貨驗證測試版

這一版重點：
1. 專門辨識「6601盤點叫貨表.xlsx」格式。
2. 讀取第 1 列品項、第 2 列日期、第 3 列「到貨日」。
3. 上傳後先顯示工作表、到貨欄、日期與逐筆預覽。
4. 日期不一致時禁止儲存，避免把錯誤欄位存入 Firebase。
5. 隔天可載入應到、輸入實到、記錄多到／少到原因。

GitHub 更新方式：
1. 先解壓縮本 ZIP。
2. 進入 GitHub：chukaikai/kitchen-calculator。
3. Add file → Upload files。
4. 將解壓縮後的所有檔案拖入，覆蓋同名檔案。
5. 按 Commit changes。
6. 等待約 1～3 分鐘，再開啟 GitHub Pages 網址。
7. 若仍看到舊畫面，請強制重新整理或用無痕視窗測試。

今天測試：
1. 首頁 → 央廚到貨驗證。
2. 到貨日期選擇叫貨表中「到貨日」對應日期。
3. 上傳 6601盤點叫貨表.xlsx。
4. 核對工作表、欄位、日期與 5～10 個品項。
5. 確認後按「確認預覽並儲存應到」。
6. 按「實到全部＝應到」或故意改一項測試差異原因。

注意：
- 這是現場驗證 Beta，第一次請不要刪除原 GitHub 檔案備份。
- 網站需要網路才能載入 Excel 解析元件與 Firebase。
